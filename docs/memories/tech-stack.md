## 通用编程原则

1. 严格禁止使用 Emoji
2. 注重模块化（多文件），单个文件代码行数不得超过 1000 行；
3. 任何代码必须包含单元测试，且测试覆盖率尽量达到 100%；
4. 任何大段的代码必须包含文档注释，且注释内容须易于理解、准确，最好使用简体中文；


## Language Preference

1. Python first（版本跟随当前运行环境，无硬性要求）
2. Shell

## 依赖管理与测试

- 包管理：`uv`
- 测试框架：`pytest`
